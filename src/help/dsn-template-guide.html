Sanitas DSN Template Guide

DSN rules customize detection without raw regex. Rules are stored only for the current browser session after you validate and apply them.

Safe Use

Use dummy examples while testing. Do not put production logs, customer data, real tokens, real emails, or real IPs into examples or screenshots.

Workflow

  1. Open the Sanitas floating window.
  2. Select the DSN Rules tab.
  3. Insert a template or the full safe example.
  4. Edit the JSON using safe literal values.
  5. Click Validate Rules.
  6. Click Apply to Current Session.
  7. Return to the Text tab and use Clean to test the rule.
  8. Click Clear when done to remove mappings, custom terms, DSN rules, and workspace text.

Templates

literal_terms

Detects exact words or phrases.

Project Titan
-> [[CUSTOM_TERM_0001]]

Options: terms, case_sensitive, match_word_boundary, placeholder_type.

labeled_value

Detects a value after a trusted label and replaces only the value.

DB_PASS=SafeDummyValue
-> DB_PASS=[[DATABASE_SECRET_0001]]

Options: labels, separators, value_mode, case_sensitive, placeholder_type.

prefix_token

Detects tokens that start with a known prefix.

sk_test_ABCDEF1234567890
-> [[API_KEY_0001]]

Options: prefixes, min_length, max_length, allowed_chars, placeholder_type.

structured_id

Detects fixed-format IDs from known prefixes and controlled segments.

DOC-INT-445566
-> [[DOCUMENT_ID_0001]]

Options: prefixes, segments, case_sensitive, placeholder_type.

context_value

Detects weak values only when a trusted context label exists.

CVV: 123
-> CVV: [[CARD_SECURITY_CODE_0001]]

Options: context_labels, separators, value_kind, min_length, max_length, placeholder_type.

Security Rules

  • No raw regex fields are allowed.
  • No JavaScript, executable fields, remote URLs, or dynamic scripts are allowed.
  • Active DSN rules are stored only in chrome.storage.session.
  • Safe summaries do not show terms, labels, prefixes, or matched values.
